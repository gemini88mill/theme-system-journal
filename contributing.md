# Contributing Guidelines

Welcome to the Theme System Journal project. This document outlines how to contribute and how contributions are reviewed.

## Table of contents
- How to contribute
- Documentation standards
- Quality checks
- Review and merge

## How to contribute
- Create issues for feature requests or bugs.
- Fork the repository (if applicable) and create a branch with a descriptive name.
- Open a pull request with a clear description of the changes.

## Documentation standards
- Use clear, concise language.
- Use Markdown with proper headings and lists.
- Include a short summary at the top of new documentation.

## Quality checks for contributions
- Markdown formatting: ensure headings, lists, and code blocks are properly used.
- Link validity: verify internal anchors and external links are reachable.
- Tests and validation: run any project-specific checks if available.

## Review and merge
- PRs will be reviewed by maintainers within a reasonable timeframe.
- Respond to feedback and iterate on changes.
- Upon approval, the maintainers will merge the PR.

## Licensing and attribution
- By contributing, you agree that your contributions are available under the project license and that attribution is given where applicable.
